Privacy Policy

Last Updated: October 28, 2025

Core Principle: PicZip IN processes images locally on your device. Image processing happens entirely offline, and processed images are stored only where you choose to save them.

1. Device Identifiers and Advertising IDs

For advertising purposes, the app may collect:

  • Google Advertising ID (GAID) - Used for personalized and non-personalized advertising
  • Application Instance ID - Anonymous identifier for app analytics

These identifiers are used solely for advertising purposes through third-party ad networks integrated via TopOn SDK. You can reset your advertising ID at any time in your device settings.

2. Runtime Permissions

The app requests the following permissions:

  • READ_EXTERNAL_STORAGE (Android 12 and below) - Required to access images from device storage
  • WRITE_EXTERNAL_STORAGE (Android 12 and below) - Required to save processed images to gallery
  • READ_MEDIA_IMAGES (Android 13+) - Required to access images using modern scoped storage
  • INTERNET - Required for loading advertisements and sending analytics data
  • ACCESS_NETWORK_STATE - Required to check network connectivity for ad loading

Permission Usage: Storage permissions are used exclusively for image selection and saving processed images. Network permissions are used only for advertisement delivery and anonymous analytics.

3. Data Categories and Purposes

3.1 Data Types Collected

  • Device Information: Device model, operating system version, screen resolution
  • Usage Data: App launches, screen views, feature usage (compression, format conversion, resizing)
  • Diagnostics: Crash reports, performance metrics, error logs
  • Advertising Interactions: Ad impressions, clicks, rewarded ad completions

3.2 Data Purposes

  • Advertising Delivery and Frequency Control: Display ads through TopOn SDK, manage ad frequency to prevent over-exposure
  • Anonymous Analytics: Understand app usage patterns, improve user experience through Firebase Analytics
  • App Functionality: Ensure core features work correctly, diagnose and fix issues via Firebase Crashlytics
  • Performance Monitoring: Track app performance metrics via Firebase Performance

4. Data Processing Location and Association

4.1 Local Processing

Image Processing: All image compression, format conversion, and resizing operations occur entirely on your device. No image data is uploaded to any server or cloud service.

4.2 Cloud Processing (Non-Image Data)

  • Analytics Data: Usage statistics, crash reports, and performance metrics are sent to Firebase servers (Google Cloud Platform)
  • Advertising Data: Ad requests and interactions are processed by TopOn SDK and ad network partners

4.3 Personal Identity Association

Analytics and advertising data are collected anonymously and are not associated with your personal identity, email address, or other personally identifiable information. Image processing data never leaves your device.

4.4 Cross-App Tracking

Advertising identifiers may be used for advertising personalization across apps. You can opt-out of personalized advertising in your device settings.

6. User Consent and Controls

6.1 Consent Mechanisms

  • GDPR/UMP Consent: For users in the European Economic Area (EEA) and United Kingdom, we implement Google's User Messaging Platform (UMP) to obtain consent for personalized advertising where required
  • iOS ATT: Not applicable (Android-only app)

6.2 Advertising Preferences

You can control advertising personalization:

  • Opt-out of personalized ads in Android Settings → Google → Ads → Reset advertising ID
  • Disable ads by purchasing the ad-free version (if available)

6.3 Withdrawal and Reset

  • Reset Advertising ID: Android Settings → Google → Ads → Reset advertising ID
  • Disable Analytics: Contact us to request analytics data deletion
  • App Deletion: Uninstalling the app removes all local data. Cloud analytics data retention follows Firebase policies.

7. Data Sharing, Retention, and Security

7.1 Data Sharing

We share data with:

  • Google (Firebase): Analytics, crash reports, performance metrics
  • TopOn SDK and Ad Networks: Advertising identifiers and interaction data for ad delivery
  • No Image Data: Image files are never shared or uploaded

7.2 Data Retention

  • Local Data: Processed images stored on device remain until manually deleted
  • Analytics Data: Firebase Analytics data is retained according to Google's data retention policies (typically 14 months)
  • Crash Reports: Firebase Crashlytics retains crash data for up to 90 days

7.3 Security Measures

  • Encryption: Image processing uses Android's secure file handling APIs
  • Sandboxing: App operates within Android's application sandbox with limited permissions
  • Minimal Permissions: App requests only necessary permissions for core functionality
  • No Backend Storage: No image data is stored on remote servers
